# Agentic Usage

Agentic usage enables AI coding assistants and developer tools to automatically understand how to use your API and SDK.

By providing structured instructions and examples, AI tools such as **Cursor, Claude Code, and GitHub Copilot** can generate correct code when developers interact with your platform.

This allows AI assistants to automatically:

- Install your SDK
- Authenticate API requests
- Send chat completion requests
- Stream responses
- Structure messages correctly
- Handle API errors

---

# What is Agentic Usage?

Agentic usage means providing documentation and skill files that AI assistants can read and interpret.

When AI coding assistants scan your project and detect these instructions, they learn how to interact with your API and can generate working integration code for developers.

This dramatically improves the quality of AI-generated code and reduces integration mistakes.

---

# Quick Start

Install the agent skill in your project.

```bash
npx add-skill openrouter-team/agent-skills
```

This installs a skill file that teaches AI coding assistants how to use your SDK and API.

---

# Install the SDK

Your project should include the official TypeScript SDK.

```bash
npm install @openrouter/sdk
```

or

```bash
yarn add @openrouter/sdk
```

---

# Import and Initialize the Client

Create a client using your API key.

```ts
import OpenRouter from "@openrouter/sdk";

const client = new OpenRouter({
  apiKey: process.env.OPENROUTER_API_KEY,
});
```

If you are running a compatible backend, you can configure a custom API endpoint.

```ts
const client = new OpenRouter({
  apiKey: process.env.OPENROUTER_API_KEY,
  baseURL: "https://your-domain.com/api/v1",
});
```

---

# Example: Basic Chat Completion

Send a simple chat completion request.

```ts
const response = await client.chat.completions.create({
  model: "gpt-4",
  messages: [
    {
      role: "user",
      content: "Hello!",
    },
  ],
});

console.log(response);
```

---

# Example: Chat Conversation

You can send multi-message conversations.

```ts
const response = await client.chat.completions.create({
  model: "gpt-4",
  messages: [
    {
      role: "system",
      content: "You are a helpful assistant",
    },
    {
      role: "user",
      content: "Explain machine learning in simple terms",
    },
  ],
});
```

---

# Streaming Responses

Streaming allows responses to be received as the model generates them.

```ts
const stream = await client.chat.completions.create({
  model: "gpt-4",
  stream: true,
  messages: [
    {
      role: "user",
      content: "Write a short story",
    },
  ],
});
```

---

# Extracting Streamed Text

To output generated text as it streams:

```ts
for await (const chunk of stream) {
  process.stdout.write(chunk.choices?.[0]?.delta?.content ?? "");
}
```

Streaming is useful for:

- Chat applications
- Real-time AI interfaces
- Token-by-token UI updates

# Using the HTTP API

AI assistants may also generate direct HTTP requests.

Example using curl:

```bash
curl https://openrouter-rsjp.onrender.com/api/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4",
    "messages": [
      {
        "role": "user",
        "content": "Explain AI in simple words"
      }
    ]
  }'
```

---

# Message Format

Messages follow a structured format.

```json
{
  "role": "user",
  "content": "Hello"
}
```

Supported roles:

- `system`
- `user`
- `assistant`

Example conversation:

```json
[
  {
    "role": "system",
    "content": "You are a helpful assistant"
  },
  {
    "role": "user",
    "content": "Explain artificial intelligence"
  }
]
```

# Error Handling

Always handle API errors in your applications.

```ts
try {
  const response = await client.chat.completions.create({
    model: "gpt-4",
    messages: [
      {
        role: "user",
        content: "Hello",
      },
    ],
  });

  console.log(response);
} catch (error) {
  console.error("API Error:", error);
}
```
